A dev layer for your codebase.

Claude Code and Codex give you powerful general-purpose agents. Archie gives you the layer they do not: isolated execution, reviewable previews, repo-local memory, and team-defined tools built around how your software actually gets built.

Worktree isolation

Each task runs in its own git worktree with its own preview and runtime context. If a run goes sideways, you throw away the branch instead of cleaning up your main codebase.

Add the tools your team actually needs

Archie ships with real workflows out of the box, but the bigger idea is extensibility. Add PR reviews, test generation, code quality checks, compliance checks, analytics validation, onboarding helpers, or anything else your team repeats.

Shared context by default

New tools inherit project memory, specs, diffs, personas, and run history instead of rediscovering the repo from scratch.

Platform reliability built in

Retries, logs, artifacts, previews, and run tracking come from Archie, so every new tool benefits from the same foundation.

Run it locally. Host it for your team.

Archie works as a local developer environment and as a shared browser-based workspace for the rest of your team.

For developers

Run it on your machine

Clone the repo, start Archie locally, and point it at your projects. Use your local tools, your local auth, and your own codebase without giving up control.

git clone https://github.com/your-org/archie
cd archie && npm install
npm run dev
For teams

Host it once, let the whole team use it

Deploy Archie on your own infrastructure and give developers, designers, PMs, and marketers a shared workspace for your codebase.

Review previews, inspect diffs, watch demos, and use team-built tools from the browser without asking everyone to install local tooling.
