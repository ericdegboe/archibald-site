Live walkthrough

Coding agents produce code, but you still have to manually click through the app to understand what changed. Archie's walkthrough lets you step through user flows with AI narration explaining each screen, so you review behavior, not just diffs.

Video demo generation

You shipped a feature in minutes, then spent an hour recording a walkthrough for stakeholders. Archie records narrated demos automatically. The agent navigates, performs actions, and produces a shareable video without you ever opening a screen recorder.

Live preview

Reading a diff doesn't tell you if the feature actually works. Each task in Archie runs its own live preview instance with a duplicated database. You visit the URL, interact with the app, and test the agent's changes before anything touches main.

Screenshot placeholder

Worktree isolation

Most AI coding setups let the agent operate directly on your main branch and production database. One bad run and you're cleaning up the mess. Archie gives every task its own git worktree and duplicated database. The worst a failed run can do is produce a branch you discard.

Stop switching tools every time something new hits Hacker News. Archie is yours to shape. Need a feature? Don't wait for a roadmap, just build it. Two ways to make it exactly what you need:

The spec drives the code.
The code updates the spec.

Every project has a living specification. Edit the spec, and Archie generates a focused task from the diff. The agent builds it in an isolated worktree. After you review, the spec updates to reflect what was actually built.

1

Edit the spec and describe what the application should do.

2

Archie generates a plain-language task summary from the diff.

3

You confirm, and the agent builds it in an isolated worktree.

4

Review the live preview, mark it complete, and the spec updates.

Behaviour

Precise, unambiguous descriptions of how the system acts. Inputs, outputs, state changes. Clear enough to hand directly to QA.

Decision

Deliberate architectural choices with rejected alternatives. Tells the agent: this was chosen consciously, build it.

Context

The reasoning behind a requirement. The agent uses it to interpret ambiguity, not as a direct instruction.

Clone it. Run it. Make it yours.

Archie is a foundation, not a prescription. Run it locally or deploy it for your whole team, including the people who never touch a terminal.

For solo developers

Run it on your machine

Clone the repo, install, and start developing. Archie runs entirely on your local machine. Your code, your data, your environment. No cloud dependency.

git clone https://github.com/your-org/archie
cd archie && npm install
npm run dev
For teams

Lovable or Base44 like for your codebase

Deploy Archie on DigitalOcean, AWS, or any server. Your entire team gets access through the browser: designers, PMs, marketing, anyone. No local setup, nothing to install, nothing to break.

Think of it as your own Lovable or Base44, but running against your actual codebase. Anyone on the team can describe a change, review a live preview, and share a demo or a PR, without writing a line of code or installing anything on their machine.
